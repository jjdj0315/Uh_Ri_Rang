# DAKER PRD (Product Requirements Document)

> 해커톤 플랫폼 — 해커톤 탐색, AI 팀매칭, 제출, 리더보드

---

## 뭘 만드는가

해커톤을 찾고, 팀을 구하고, 결과물을 제출하는 플랫폼이다. 핵심은 **AI가 팀을 매칭해주는 것**.

---

## 사용자는 누구인가

| 역할 | 하고 싶은 것 |
|------|------------|
| 무소속 | 참가할 해커톤 찾기, 들어갈 팀 찾기 |
| 팀장 | 팀에 맞는 팀원 찾기 |
| 팀원 | 내 팀 현황 확인, 리더보드 확인 |

첫 방문 시 역할을 선택하면 이후 Camp 페이지 UI가 역할에 맞게 바뀐다.

---

## 페이지 5개, 각각 하는 일

### 1. 메인 (`/`)
진행 중인 해커톤을 카드로 보여주고, 해커톤 목록/Camp/랭킹으로 보낸다.
첫 방문이면 역할 선택 다이얼로그가 뜬다.

### 2. 해커톤 목록 (`/hackathons`)
전체 해커톤을 카드로 나열한다. 상태(진행중/종료/예정), 태그로 필터링.
카드 클릭하면 상세 페이지로 이동.

### 3. 해커톤 상세 (`/hackathons/:slug`)
하나의 해커톤에 대한 모든 정보. 탭 8개:

| 탭 | 하는 일 |
|----|--------|
| 개요 | 대회 설명, 팀 정책 |
| 안내 | 공지사항, 규정/FAQ 링크 |
| 평가 | 평가 기준 (점수형 or 투표형) |
| 일정 | 마일스톤 타임라인 |
| 상금 | 순위별 상금 |
| Teams | 이 해커톤의 팀 목록, Camp으로 이동 |
| Submit | 제출물 업로드 (제목/파일/URL/메모) |
| Leaderboard | 순위 테이블 |

### 4. 팀원 모집 (`/camp`)
역할에 따라 다르게 동작한다:

- **무소속** → 모집 중인 팀 리스트 + "AI로 팀 찾기"
- **팀장/팀원** → 예비 팀원 리스트 + "AI로 팀원 찾기"

팀 만들기(무소속만), 연락하기, AI 검색 가능.

### 5. 랭킹 (`/rankings`)
전체 해커톤 통합 순위. Top 3 포디엄 카드, 내 팀 하이라이트, 기간 필터.

---

## 핵심 기능 3가지

### 1. AI 팀매칭
자연어로 "Python 백엔드인데 기획자 있는 팀 찾아줘" 입력하면 Gemini가 팀을 추천하고 이유를 알려준다. 역할에 따라 팀을 찾아주거나 팀원을 찾아준다.

### 2. 역할 시스템
첫 방문 시 팀장/팀원/무소속을 선택. 이후 Camp 페이지 UI, AI 검색 프롬프트, 리더보드 하이라이트가 역할에 맞게 바뀐다.

### 3. 해커톤 상세 + 제출
하나의 페이지에서 대회 정보 확인부터 제출까지 전부 가능. 탭으로 전환하며 개요/일정/상금/팀/제출/리더보드를 본다.

---

## 데이터는 어떻게 저장하나

**백엔드 없음. localStorage가 DB.**

| 키 | 내용 | 누가 쓰나 |
|----|------|----------|
| hackathons | 해커톤 목록+상세 | 초기화 시 세팅 |
| teams | 팀 목록 | Camp에서 팀 생성 |
| submissions | 제출물 | Submit 탭에서 제출 |
| leaderboards | 리더보드 | 초기화 시 세팅 |
| userProfile | 역할 정보 | 첫 방문 시 선택 |

앱 최초 로드 시 예시 JSON으로 자동 초기화. 손상되면 자동 복구.

---

## 유일한 서버 로직

```
/api/match (Vercel 서버리스)
```

Gemini API 키를 숨기기 위해 존재. 브라우저 → /api/match → Gemini → 결과 반환.
나머지는 전부 브라우저에서 localStorage로 처리.
