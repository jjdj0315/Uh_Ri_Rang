🔑 왜 서버를 거치나요?

Gemini API 키를 브라우저에 넣으면 누구나 개발자도구에서 볼 수 있습니다. /api/match라는 서버(Vercel 서버리스 함수)를 중간에 두면, 키는 서버 환경변수에만 존재하고 브라우저에는 절대 노출되지 않습니다.

03

데이터 흐름

앱 최초 로드
localStorage에 데이터가 있는지 확인
데이터 없으면 → 초기화
예시 JSON 파일 (data/예시자료/*.json)을 localStorage에 복사
localStorage.setItem("hackathons", JSON.stringify(sampleData))
데이터 있으면 → 바로 렌더링
각 페이지에서 localStorage를 읽어 화면에 표시
const hackathons = JSON.parse(localStorage.getItem("hackathons"))
사용자 액션 시 → 저장
팀 생성, 제출물 업로드 등은 localStorage에 직접 쓰기
localStorage.setItem("teams", JSON.stringify([...teams, newTeam]))

05

AI 팀매칭 흐름 (Track 1)

1
사용자가 자연어로 입력
"React 프론트 개발자 있는 팀 찾아줘"
📍 /camp 페이지의 AI 검색창
2
브라우저 → API Route 요청
검색어 + localStorage의 teams 데이터를 함께 POST 전송
📍 POST /api/match { query, teams[] }
3
서버에서 Gemini API 호출
프롬프트: "다음 팀 목록에서 사용자 조건에 맞는 팀을 추천하고 근거를 설명해줘"
📍 Vercel 서버리스 → Gemini API (키는 서버 환경변수)
4
Gemini가 추천 결과 반환
매칭된 팀 + 추천 이유를 JSON으로 반환
📍 { matches: [{ teamName, reason, score }] }
5
사용자에게 결과 표시
추천 팀 카드 + "이 팀을 추천하는 이유" 함께 렌더링
📍 /camp 페이지에 결과 UI
06

핵심 요약

항목 설명
Backend 없음. localStorage가 DB 역할
Server Code /api/match 하나만 존재 (Gemini API 키 숨김용)
Data Init 앱 최초 로드 시 예시 JSON → localStorage 복사
State React state + localStorage (별도 상태 라이브러리 없음)
Deploy Vercel (정적 페이지 + API Route 서버리스)
AI Feature 자연어 → Gemini → 팀 추천 + 근거
